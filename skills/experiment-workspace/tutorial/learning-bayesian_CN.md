> 本教程通过阅读 `experiment-workspace` 中的 `analyze-bayesian.py` 和 `references/analysis-bayesian.md` 逐步学习贝叶斯 A/B 测试的实现逻辑。
> 记录了学习过程中的真实问答，作为持续学习的 context。

## 第一章：为什么用 Bayesian？

传统 A/B 测试（频率派）问的是反事实问题：

> "假设 treatment **没有效果**，我观察到这个数据的概率是多少？"

这就是 p-value。它回答的是一个间接问题，不直观。

Bayesian 直接回答：

> "看完这批数据之后，treatment **真的更好**的概率是多少？"

输出就是 `P(win) = 94%`，意思是：基于当前数据，有 94% 的概率 treatment 确实更好。可以直接用于决策。

## 第二章：两种指标类型

### 判断标准

看你的原始数据是什么形状：

| 问题 | 指标类型 | 例子 |
|------|---------|------|
| 每个用户只有"发生"或"没发生" | 比例指标（proportion） | 点击了没有、注册了没有、购买了没有 |
| 每个用户有一个具体数值 | 连续指标（continuous） | 花了多少钱、停留了多少秒、看了几篇文章 |

### 输入数据格式

```json
// 比例指标：只需要 n 和 k
"cta_clicked": {
  "control":   {"n": 5000, "k": 300},
  "treatment": {"n": 5050, "k": 350}
}

// 连续指标：需要 n、sum、sum_squares
"revenue_per_user": {
  "control":   {"n": 5000, "sum": 45000.0,  "sum_squares": 820000.0},
  "treatment": {"n": 5050, "sum": 48020.0,  "sum_squares": 901000.0}
}
```

### sum_squares 是什么？

`sum_squares` 是每个用户的值**各自平方后再加总**：

```
sum_squares = x₁² + x₂² + x₃² + ... + xₙ²
```

注意：**不是** `(x₁ + x₂ + ... + xₙ)²`，不是 sum 的平方。

**为什么需要它？** 因为方差的公式是：

```
var = (sum_squares - sum²/n) / (n - 1)
```

只有 `sum` 算不出方差，必须有 `sum_squares`。实际项目里数据在数据库里，直接聚合查询即可：

```sql
SELECT
  COUNT(*)            AS n,
  SUM(value)          AS sum,
  SUM(value * value)  AS sum_squares
FROM events
WHERE variant = 'control'
```

一条 SQL，三个数字，脚本拿到这三个数就能算出均值和方差。

### 边界情况：inverse 指标

错误率、延迟等"越低越好"的指标，加 `"inverse": true`，P(win) 的含义变为"treatment 的值更低的概率"。

## 第三章：所有变量的定义与计算

### 3.1 比例指标案例

**场景：** 测试新版 CTA 按钮是否能提升点击率。

**原始数据：**

| 变体 | 曝光人数 n | 点击人数 k |
|------|-----------|-----------|
| control | 5000 | 300 |
| treatment | 5050 | 350 |

**mean（均值）= 转化率：**

```
mean_a = 300 / 5000 = 0.06     （6.00%）
mean_b = 350 / 5050 = 0.06931  （6.93%）
```

**var（方差）= 单个用户行为的离散程度：**

对比例指标，每个用户非 0 即 1，方差由均值本身决定：

```
var_a = 0.06 × (1 - 0.06)      = 0.0564
var_b = 0.06931 × (1 - 0.06931) = 0.06450
```

均值越接近 0.5，方差越大（结果最不确定）；越接近 0 或 1，方差越小。

**μ_rel（相对效应点估计）= 观测到的相对提升：**

```
μ_rel = (mean_b - mean_a) / mean_a
      = (0.06931 - 0.06) / 0.06
      = +15.52%
```

**se（标准误差）= 对 μ_rel 这个估计有多不确定：**

用 Delta Method 计算：

```
se = sqrt(
    var_b / (n_b × mean_a²)
  + var_a × mean_b² / (n_a × mean_a⁴)
)
≈ 8.79%
```

se 越大 = 样本越少或方差越大 = 估计越不可靠。
分母里有 `mean_a²`，所以 baseline 转化率越低，se 越大——低转化率实验天然更不确定。

---

### 3.2 连续指标案例

**场景：** 测试新结账流程是否提升每用户收入。

**原始数据：**

| 变体 | n | sum | sum_squares |
|------|---|-----|-------------|
| control | 5000 | 45000.0 | 820000.0 |
| treatment | 5050 | 48020.0 | 901000.0 |

**计算过程：**

```
# control
mean_a = 45000 / 5000 = 9.0（元/人）
var_a  = (820000 - 45000²/5000) / (5000-1)
       = (820000 - 810000) / 4999
       ≈ 2.0004

# treatment
mean_b = 48020 / 5050 ≈ 9.509（元/人）
var_b  = (901000 - 48020²/5050) / (5050-1)
       ≈ 88.01
```

注意：连续指标的 var 通常比比例指标大很多（用户间收入差异大），但 n 足够大时 se 仍然可以很小。

```
μ_rel = (9.509 - 9.0) / 9.0 = +5.65%
se    ≈ 1.64%

P(win) = norm.sf(0, 0.0565, 0.01637) ≈ 99.97%
```

## 第四章：后验分布 N(μ_rel, se²)

### N 是什么？

`N` 是**正态分布（Normal / Gaussian Distribution）**。记号 `N(μ, σ²)` 中：

- `μ` = 均值（最可能的值）
- `σ²` = 方差，`σ = se` = 标准差（不确定程度）

注意：`N(μ, σ²)` 的第二个参数是方差（σ²），不是标准差（σ）。代码里操作的是 `se`（即 σ），因为标准差和原始数据单位相同，更直观。

### 为什么是分布而不是一个数？

因为你**不知道真实的 δ 是多少**，只能说它"大概在哪个范围"。

类比：朋友说"我大概 3 点到，但可能早或晚半小时"——给你的不是一个确定时间，而是一个带不确定性的估计。正态分布就是这种不确定性的数学表达：

```
μ = 最可能的值（3点）
σ = 不确定程度（半小时）
```

在 A/B 测试里，观测到的 δ = +15.52% 只是一次采样结果。如果再跑一次实验，数据略有不同，δ 也会略有不同。正态分布描述的是：**真实的 δ 落在各个位置的可能性有多大**。

### 为什么能用正态分布近似？

来自**中心极限定理（CLT）**：

> 当样本量足够大时，样本均值的分布趋近于正态分布——无论原始数据是什么分布。

这也是为什么脚本要求每个变体至少 **30 次转化**——低于这个数，CLT 还没生效，正态近似不可靠，P(win) 和 risk 的值可能是误导性的。

## 第五章：P(win) 的计算

### 直觉

`P(win) = δ > 0 的概率 = 后验曲线中 0 右侧的面积`

```
         ▲
         │         ┌───┐
         │       ┌─┘███└─┐
         │      ─┘███████└─
         │   ───████████████───
         └─────┬───────────────▶ δ（相对提升）
               0   +μ_rel
         ←────→←──────────────→
          P(ctrl好)  P(win)
```

因为均值 +15.52% 在 0 的右边，右侧面积（P(win)）会很大。

### 代码实现

```python
ctw = norm.sf(0.0, loc=μ_rel, scale=se)
```

`sf` = survival function = `1 - CDF`，即**大于某个值的概率**：

```
norm.sf(0, loc=0.1552, scale=0.0879)
= P(δ > 0)，其中 δ ~ N(0.1552, 0.0879²)
= P(treatment 的真实提升 > 0)
= P(win)
≈ 96.1%
```

均值离 0 约 1.76 个标准差（0.1552 / 0.0879），所以概率很高。

## 第六章：这些计算和贝叶斯的关系

很多人会困惑：上面的计算看起来都是"频率派"的工具（均值、方差、标准误差），贝叶斯在哪里？

**贝叶斯定理：**

```
后验 ∝ 先验 × 似然
```

**默认模式（flat prior，`proper: false`）：**

先验是"平坦的"——对所有可能的 δ 值一视同仁，没有任何偏好。这时候：

```
后验 ∝ 1 × 似然 = 似然
```

后验完全等于数据的似然函数。`N(μ_rel, se²)` 是对似然函数的高斯近似——当样本足够大时（CLT），似然函数形状近似正态。

所以整个计算流程是：

```
观测数据
  → 算出点估计 μ_rel 和不确定性 se    （频率派的工具）
  → 用 N(μ_rel, se²) 近似后验         （贝叶斯的框架）
  → 从后验里算 P(win)、CI、risk        （贝叶斯的输出）
```

**贝叶斯的价值在于框架，不在于计算方法。** 它让你能说出"P(win) = 96%"这句话——在频率派里，你不能直接说"treatment 更好的概率是 96%"，只能说"p < 0.05，拒绝零假设"。

**启用先验（`proper: true`）时，贝叶斯才真正"介入"计算：**

```
post_mean = (data_mean/data_var + prior_mean/prior_var)
          / (1/data_var + 1/prior_var)
post_std  = sqrt(1 / (1/data_var + 1/prior_var))
```

先验会把后验均值拉向历史经验。样本小时先验影响大，样本大时数据压制先验——这是纯频率派做不到的。
